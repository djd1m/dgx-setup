# 01. Claude Code на локальной Ollama — рецепт для AI-кодера

Цель: установить Claude Code и направить его на Anthropic-совместимый эндпоинт локальной Ollama (`http://localhost:11434`) — без аккаунта, ключа и запросов на серверы Anthropic.

Интеграция официально описана Ollama: [docs/integrations/claude-code.mdx](https://github.com/ollama/ollama/blob/main/docs/integrations/claude-code.mdx).

---

## Предусловия

Выполнена инструкция [00-ollama.md](00-ollama.md). Проверить до первого шага:

```bash
ollama list
```

Ожидаемый результат: таблица со скачанными моделями; целевая модель (`qwen3.6:27b`) присутствует в колонке `NAME`.
Если не так, то: остановиться, вернуться к 00-ollama.md. Модель, которой нет в `ollama list`, использовать нельзя.

```bash
ollama ps
```

Ожидаемый результат: команда отрабатывает без ошибки соединения (Ollama слушает `localhost:11434`); колонка `CONTEXT` у загруженной модели — `64000`, т.е. `OLLAMA_CONTEXT_LENGTH=64000` из инструкции 00 применился.
Если не так, то: остановиться, вернуться к 00-ollama.md. Малый контекст — основная причина того, что Claude Code «теряет нить» и забывает файлы.

---

## Переменные

| Переменная | Значение | Смысл |
|---|---|---|
| `ANTHROPIC_BASE_URL` | `http://localhost:11434` | куда слать запросы. Вместо Anthropic — на локальную Ollama |
| `ANTHROPIC_AUTH_TOKEN` | `ollama` | Ollama ключ не проверяет, но переменная нужна непустая |
| `ANTHROPIC_API_KEY` | `""` | **пустой намеренно** — именно это отключает поход в Anthropic |
| `ANTHROPIC_MODEL` | `qwen3.6:27b` | имя должно **точно** совпадать с выводом `ollama list` |

Значения — дословно из [документации Ollama](https://github.com/ollama/ollama/blob/main/docs/integrations/claude-code.mdx).

---

## Шаги

### Шаг 1. Установить Claude Code

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Официальный способ — [Advanced setup](https://code.claude.com/docs/en/setup).

Ожидаемый результат: скрипт скачался и отработал.

Если не так, то (ресурс Anthropic, из России может быть недоступен; *достоверно проверить из российской сети не удалось — **NOT VERIFIED***) — ветка npm, её документирует сама Anthropic, и npm из России работает:

```bash
npm install -g @anthropic-ai/claude-code
```

Требуется Node.js 22 или новее. Пакет ставит **тот же самый бинарник**, что и скрипт.

> ⚠️ **`sudo npm install -g` не использовать** — прямое предупреждение из документации Anthropic: приводит к проблемам с правами и рискам безопасности.

Если не отработала и ветка npm: остановиться, спросить человека. См. «Стоп-условия».

### Шаг 2. Проверить установку

```bash
claude --version
```

Ожидаемый результат: напечатана версия.
Если не так, то: вернуться к шагу 1 и выполнить вторую ветку (npm).

### Шаг 3. Направить Claude Code в Ollama

```bash
export ANTHROPIC_AUTH_TOKEN=ollama
export ANTHROPIC_API_KEY=""
export ANTHROPIC_BASE_URL=http://localhost:11434
```

Закрепить, чтобы не вводить каждый раз:

```bash
echo 'export ANTHROPIC_AUTH_TOKEN=ollama' >> ~/.bashrc
echo 'export ANTHROPIC_API_KEY=""' >> ~/.bashrc
echo 'export ANTHROPIC_BASE_URL=http://localhost:11434' >> ~/.bashrc
source ~/.bashrc
```

Ожидаемый результат:

```bash
echo $ANTHROPIC_BASE_URL
```

→ `http://localhost:11434`

Если не так, то: проверить, что строки дописались в `~/.bashrc` без дублей, и повторить `source ~/.bashrc`.

### Шаг 4. Выбрать модель

```bash
export ANTHROPIC_MODEL=qwen3.6:27b
```

Ожидаемый результат: значение совпадает с именем из `ollama list` посимвольно.
Если не так, то: взять имя из `ollama list` и подставить его. Не угадывать имя.

### Шаг 5. Запустить

```bash
cd ~/твой-проект
claude
```

Либо короткий путь от самой Ollama — она поднимет всё сама:

```bash
ollama launch claude
```

Для скриптов и CI (флаг `--yes` пропускает выбор модели и скачивает её, если нужно):

```bash
ollama launch claude --model qwen3.6:27b --yes -- -p "твой запрос"
```

Ожидаемый результат: Claude Code запускается и отвечает на простой вопрос; `ollama ps` во время работы показывает загруженную модель; в `nvidia-smi` при запросе видна нагрузка на GPU.
Если не так, то: выполнить curl-проверку из раздела «Критерий готовности» — она отделяет проблему Claude Code от проблемы Ollama.

## Ограничения эндпоинта

Ollama реализует Anthropic-протокол не полностью. Список из [anthropic-compatibility.mdx](https://github.com/ollama/ollama/blob/main/docs/api/anthropic-compatibility.mdx). **Не отлаживать это — оно не сломано, оно не реализовано.**

| Чего нет | Насколько мешает |
|---|---|
| Prompt caching | заметно: каждый запрос считается заново, медленнее и дороже по времени |
| `tool_choice` | нельзя заставить вызвать конкретный инструмент |
| `/v1/messages/count_tokens` | подсчёт токенов недоступен |
| Batches API | пакетная обработка |
| Citations, PDF | работа с документами |
| Картинки по URL | только base64 |

Две оговорки из доков: подсчёт токенов **приблизительный**, а `budget_tokens` у thinking **принимается, но не соблюдается**.

Работает: сообщения, стриминг, системные промпты, многоходовые диалоги, vision (картинки), **вызов инструментов (tools)**, результаты инструментов, thinking-блоки. Вызов инструментов — критично: без него агент не читает файлы и не запускает команды.

## Стоп-условия

Не делать. Остановиться и спросить человека:

| Ситуация | Почему стоп |
|---|---|
| `sudo npm install -g @anthropic-ai/claude-code` | прямо запрещено документацией Anthropic: права и безопасность |
| `ANTHROPIC_API_KEY` заполнить непустым значением | пустой — намеренно; непустой включает поход в Anthropic |
| Подставить модель, которой нет в `ollama list` | имя должно совпадать точно |
| Не отработали обе ветки шага 1 (install.sh и npm) | дальше — догадки |
| Не выполнена 00-ollama.md / `ollama ps` не отвечает | фундамента нет |
| Чинить prompt caching, `tool_choice`, `count_tokens`, batches, citations/PDF, картинки по URL | не реализовано в эндпоинте, а не сломано |
| Вопрос о правилах Anthropic (см. ниже) | решает человек, не агент |

**Про правила Anthropic.** В системных требованиях Claude Code, наравне с ОС и памятью, указано местоположение — [Advanced setup](https://code.claude.com/docs/en/setup) ссылается на [список поддерживаемых стран](https://www.anthropic.com/supported-countries), России в нём нет. В этой конфигурации обращения к сервисам Anthropic нет вообще: программа работает как локальный клиент к твоей Ollama, и такую связку **официально документирует сама Ollama**. Тем не менее требование в документации сформулировано общо — это решение человека, не агента.

**Это не Claude.** Запускается Qwen или GPT-OSS в оболочке Claude Code: интерфейс тот же, модель другая и слабее. Агентный цикл (прочитать файл → подумать → отредактировать → проверить) требователен к качеству вызова инструментов, локальные модели тут заметно уступают. Больше ошибок, задачи проще — это ожидаемо, не баг.

## Критерий готовности

- [ ] `claude --version` печатает версию
- [ ] `echo $ANTHROPIC_BASE_URL` → `http://localhost:11434`
- [ ] `claude` запускается и отвечает на простой вопрос
- [ ] `ollama ps` во время работы показывает загруженную модель
- [ ] в `nvidia-smi` при запросе видна нагрузка на GPU

Финальная проверка связки одной командой, без запуска Claude Code:

```bash
curl -s http://localhost:11434/v1/messages \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen3.6:27b","max_tokens":64,"messages":[{"role":"user","content":"Скажи привет"}]}'
```

Ожидаемый результат: JSON с ответом модели. Пришёл — всё сходится.
